# Customer Churn Prediction

This project is an end-to-end Machine Learning solution for predicting customer churn. It includes a training pipeline with MLflow for tracking experiments, and a FastAPI-based web service to serve the trained model for inference.

## Project Structure

- `main.py`: The machine learning training script. It loads the `customer.csv` dataset, preprocesses it, and trains a `RandomForestClassifier`. It runs multiple experiments across different tree depths, logs metrics and artifacts to MLflow, and saves the best performing model.
- `app.py`: A FastAPI web application that loads the trained model and provides an API endpoint for predicting churn.
- `customer.csv`: The dataset used for training the model.
- `customer_churn_model.joblib`: The serialized final model, generated by `main.py`.

## Getting Started

### Prerequisites

Ensure you have your virtual environment activated and the necessary dependencies installed (e.g., pandas, scikit-learn, mlflow, fastapi, uvicorn, joblib).

```bash
source ../myenv/bin/activate
# Make sure to install dependencies if you haven't already
```

### 1. Training the Model

To train the machine learning model, run the `main.py` script. This script will preprocess the dataset, train several random forest models, log the results to MLflow, and export the best model to a `joblib` file.

```bash
python main.py
```

You can view the MLflow UI to analyze the training experiments by running:

```bash
mlflow ui
```

### 2. Running the API

Once the model is trained, you can start the FastAPI application to serve the model:

```bash
uvicorn app:app --reload
```

The server will start locally, usually at `http://127.0.0.1:8000`. You can visit `http://127.0.0.1:8000/docs` to interact with the API through the automatic Swagger UI documentation.
